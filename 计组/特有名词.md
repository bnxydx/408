# 存储

## MAR和MDR

1.   **MAR:存储器地址寄存器**
2.   **MDR:存储器数据寄存器**

**寄存器： 是 CPU 内部用来暂时存放指令、数据和地址的**极小容量、极高速度**的存储单元。**

## RAM和ROM

RAM (Random Access Memory) —— 随机存取存储器

ROM (Read-Only Memory) —— 只读存储器

| **概念** | **全称**         | **位置**      | **主要作用**                   |
| -------- | ---------------- | ------------- | ------------------------------ |
| **MAR**  | 存储器地址寄存器 | CPU 内部      | 指向“在哪读写”                 |
| **MDR**  | 存储器数据寄存器 | CPU 内部      | 存放“读写什么”                 |
| **RAM**  | 随机存取存储器   | 主存/缓存     | 运行程序时的临时仓库（断电丢） |
| **ROM**  | 只读存储器       | 主板/存储芯片 | 存放系统启动指令（断电不丢）   |

---

当你运行一个程序时，程序代码存在 **ROM** 或硬盘里，运行后加载到 **RAM**；CPU 在执行每一条指令时，通过 **MAR** 找到地址，通过 **MDR** 搬运具体的数据内容。

# 指令

### 1. 字节（Byte）：永恒不变的“硬标准”

-   **定义**：1个字节固定等于 **8个二进制位（bit）**。

    $$1 \text{ Byte} = 8 \text{ bits}$$

-   **特性**：无论是在老旧的 8 位单片机、过去的 32 位系统，还是现在的 64 位处理器中，**1 字节永远是 8 位**。它是计算机微观世界里的“标准度量衡”（就像国际单位制里的“克”或“米”）。

### 2. 字（Word）：因机器而异的“软标准”

-   **定义**：**“字”是指 CPU 在单位时间内一次性能够处理的二进制数据的规模。** 这个规模的长度被称为**字长（Word Length）**。
-   **特性**：字的大小是**随着 CPU 架构的变化而变化**的。
    -   在 **16位** 处理器中：1 个字 = 16 位 = 2 字节
    -   在 **32位** 处理器中：1 个字 = 32 位 = 4 字节
    -   在 **64位** 处理器中：1 个字 = 64 位 = 8 字节
